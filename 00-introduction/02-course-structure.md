# Course Structure & Roadmap

## Topics We'll Cover (In Order)

### Phase 1: Foundations
1. **OOP Concepts** — Classes, Objects, Inheritance, Polymorphism, Abstraction, Encapsulation
2. **SOLID Principles** — SRP, OCP, LSP, ISP, DIP

### Phase 2: Patterns
3. **Creational Patterns** — Singleton, Factory, Abstract Factory, Builder, Prototype
4. **Structural Patterns** — Adapter, Composite, Proxy, Decorator, Facade, Flyweight, Bridge
5. **Behavioral Patterns** — Observer, Strategy, Command, State, Template Method, Chain of Responsibility, Iterator, Mediator, Visitor

### Phase 3: Communication & Application
6. **UML Diagrams** — Class diagrams, Sequence diagrams, Activity diagrams
7. **Case Studies** — Real-world system designs implemented with everything above

## How Each Topic Will Be Covered

For every concept:
1. **What** it is (theory, definition)
2. **Why** it exists (what problem it solves)
3. **When** to use it (and when NOT to)
4. **How** to implement it (code)
5. **Trade-offs** (nothing is free — every pattern has a cost)

## The Rule

No moving to the next topic until the current one is solid. Half-understood concepts compound into complete confusion later.
